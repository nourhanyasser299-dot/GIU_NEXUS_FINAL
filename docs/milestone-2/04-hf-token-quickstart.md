# Hugging Face: Permanent Token, Code Integration, and Demo

This guide takes a developer on `HusseinSelim-1977/Nourhan_Project` from "no token" to a working three-endpoint demo. Every command and path below is verified against the current `feature/hf-job-summarization` branch.

## Step 1 — Generate a permanent token

Hugging Face access tokens are long-lived by default — they stay valid until you explicitly revoke them. Treat that as "permanent."

- Sign in at https://huggingface.co.
- Go to https://huggingface.co/settings/tokens → **+ New token**.
- **Type:** *Read*. The three endpoints (`/api/ai/skills/extract`, `/api/ai/match`, `/api/ai/summarize`) only call `POST /models/...` for inference — they never write.
- **Name:** `giu-nexus-backend`.
- Click **Generate**, then **Copy** once. Hugging Face will not show the token again. Paste it straight into your password manager — never into chat, tickets, screenshots, or commit messages.

## Step 2 — Insert the token into the code

The backend reads the token from `process.env.HF_API_TOKEN` only. From `Backend/src/services/huggingface.client.js`:

```js
const token = process.env.HF_API_TOKEN || '';
if (token) headers.Authorization = `Bearer ${token}`;
```

Wire it in:

```bash
cd Backend
cp .env.example .env   # skip if .env already exists
```

Edit `Backend/.env`:

```
HF_API_TOKEN=hf_your_new_token_here
HF_API_BASE=https://api-inference.huggingface.co
HF_SKILL_MODEL=jjzha/jobbert_skill_extraction
HF_MATCH_MODEL=sentence-transformers/all-MiniLM-L6-v2
HF_SUMMARY_MODEL=sshleifer/distilbart-cnn-12-6
```

`Backend/.gitignore` already lists `.env`, so the file stays out of git. Confirm before committing anything:

```bash
git status --ignored | grep -F .env   # .env must show as ignored
```

## Step 3 — Run the demo

```bash
# Terminal 1
cd Backend && npm install && npm run dev   # Express on http://localhost:5000
```

```bash
# Terminal 2
cd Frontend && npx http-server . -p 8080
```

Smoke-test the token from the shell first:

```bash
curl -s http://localhost:5000/api/ai/skills/extract \
  -H 'Content-Type: application/json' \
  -d '{"text":"Built React + Node.js dashboards with Postgres and AWS Lambda."}' | jq
```

A working response has `"source": "huggingface"` and a non-empty `model` field. If it says `"source": "heuristic"`, the token did not reach the service — recheck Step 2.

Then open http://localhost:8080:

1. Register or log in.
2. **Seeker dashboard** → paste a CV → **Extract Skills**. Status should read *"Generated by jjzha/jobbert_skill_extraction."*.
3. **Jobs board** → paste a CV → **Score Jobs**. Each card grows a gold "X% Match" badge plus a progress bar.
4. Click a card → **Generate Summary**. Source line names the DistilBART model.

## Troubleshooting

- **401 Unauthorized** — token typo, or the token lacks inference permission. Re-mint as *Read*.
- **503 "Model is currently loading"** — free-tier cold start. The client passes `wait_for_model: true`; rerun after ~20 s.
- **`source: "heuristic" / "jaccard" / "extractive"` in any response** — the backend never saw `HF_API_TOKEN`. Confirm `Backend/.env` is in the same directory you ran `npm run dev` from.
- **`.env` accidentally staged** — `git restore --staged Backend/.env` before recommitting.

That's the full path: token in your password manager, value in `Backend/.env`, three endpoints answering with real Hugging Face source attributions. You're set to run the demo.
